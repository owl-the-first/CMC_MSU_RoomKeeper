# CMC MSU RoomKeeper Bot

**CMC MSU RoomKeeper Bot** — Telegram-бот для поиска и бронирования свободных аудиторий на факультете ВМК МГУ.

Проект скачивает PDF-файлы расписания с сайта ВМК, извлекает из них занятость аудиторий, сохраняет данные в SQLite-базу и позволяет пользователю через Telegram-бота:

- искать свободные аудитории по дате и времени;
- фильтровать аудитории по номеру;
- создавать заявки на бронирование;
- смотреть свои заявки;
- отменять свои заявки.

## Возможности проекта

- загрузка расписания с официальной страницы ВМК;
- поиск PDF-файлов с расписанием;
- парсинг занятых аудиторий из PDF;
- хранение аудиторий, занятий и заявок в SQLite;
- поиск свободных аудиторий с учётом:
  - официального расписания;
  - уже созданных заявок на бронирование;
  - чётной/нечётной недели;
- Telegram-команды для работы с ботом;
- локализация интерфейса бота (русский и английский, Babel);
- модерация заявок администраторами;
- тесты, проверка покрытия, линтер и Sphinx-документация.

## Установка

Склонировать репозиторий:

```bash
git clone https://github.com/owl-the-first/CMC_MSU_RoomKeeper.git
cd CMC_MSU_RoomKeeper
```

Создать и активировать виртуальное окружение:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Установить проект вместе с зависимостями для разработки:

```bash
python3 -m pip install ".[dev]"
```

Для разработки с автоматическим подхватом изменений в исходниках можно
использовать editable-установку (`pip install -e ".[dev]"`), но на Python 3.12
она может работать нестабильно — в этом случае переустановите пакет обычной
командой выше.

После установки будут доступны консольные команды:

```bash
roomkeeper-init-db
roomkeeper-update-schedule
roomkeeper-import-schedule
roomkeeper-db-stats
roomkeeper-bot
```

## Настройка окружения

Для запуска Telegram-бота нужен токен бота.

Скопировать пример файла окружения:

```bash
cp .env.example .env
```

В файле `.env` указать свой токен:

```env
TELEGRAM_BOT_TOKEN=your_telegram_bot_token_here
ROOMKEEPER_DATABASE_URL=sqlite:///data/roomkeeper.db
ROOMKEEPER_DEFAULT_LOCALE=ru
TELEGRAM_ADMIN_IDS=123456789
```

Переменные:

- `TELEGRAM_BOT_TOKEN` — токен бота от [@BotFather](https://t.me/BotFather).
- `ROOMKEEPER_DATABASE_URL` — URL базы данных (по умолчанию локальный SQLite).
- `ROOMKEEPER_DEFAULT_LOCALE` — язык интерфейса по умолчанию (`ru` или `en`).
- `TELEGRAM_ADMIN_IDS` — числовые Telegram ID администраторов через запятую.
  Администраторы получают уведомления о новых заявках и могут их одобрять
  или отклонять. Узнать свой ID можно у [@userinfobot](https://t.me/userinfobot).

По умолчанию используется локальная SQLite-база:

```text
data/roomkeeper.db
```

## Подготовка базы данных и расписания

Создание таблицы базы данных:

```bash
roomkeeper-init-db
```

Далее нужно скачать PDF-файлы расписания и распарсить их в JSON:

```bash
roomkeeper-update-schedule
```

После этого появятся локальные данные:

```text
data/raw/
data/parsed/schedule_slots.json
```

Далее импортировать расписание в базу данных:

```bash
roomkeeper-import-schedule
```

По желанию проверить содержимое базы:

```bash
roomkeeper-db-stats
```

## Запуск Telegram-бота

После настройки `.env`, создания базы и импорта расписания запуск бота:

```bash
roomkeeper-bot
```

Остановить бота можно сочетанием клавиш:

```text
Ctrl+C
```

## Команды бота

* `/start` --> Запуск бота.

* `/help` --> Показ доступных команд.

* `/free` --> Поиск свободных аудиторий.

Формат для `/free`:

```text
/free ДАТА НАЧАЛО КОНЕЦ [ЧЁТНОСТЬ_НЕДЕЛИ] [АУДИТОРИЯ]
```

Примеры:

```text
/free 2026-06-22 08:45 10:20
/free 2026-06-22 08:45 10:20 even
/free 2026-06-22 08:45 10:20 odd
/free 2026-06-22 08:45 10:20 605
/free 2026-06-22 08:45 10:20 even 605
```

Чётность недели:

```text
all  — любая неделя
even — чётная неделя
odd  — нечётная неделя
```

* `/book` --> Создаёт заявку на бронирование аудитории.

Формат для `/book`:

```text
/book АУДИТОРИЯ ДАТА НАЧАЛО КОНЕЦ [ЧЁТНОСТЬ_НЕДЕЛИ] ЦЕЛЬ
```

Примеры:

```text
/book 605 2026-06-22 08:45 10:20 подготовка к защите
/book 605 2026-06-22 08:45 10:20 even подготовка к защите
/book П-13 2026-06-22 12:50 14:25 odd встреча команды
```

Если аудитория занята по расписанию или уже забронирована другим пользователем, то заявка не создаётся.

* `/my_bookings` --> Показ последних заявок текущего пользователя.

* `/cancel_booking` --> Отмена заявки пользователя по номеру.

Формат для `/cancel_booking`:

```text
/cancel_booking НОМЕР_ЗАЯВКИ
```

Пример:

```text
/cancel_booking 3
```

Короткий вариант:

```text
/cancel 3
```

* `/lang` --> Выбор языка интерфейса бота.

Формат:

```text
/lang ru
/lang en
```

Без аргументов команда показывает подсказку. Язык по умолчанию задаётся
переменной `ROOMKEEPER_DEFAULT_LOCALE` в `.env`.

## Команды администратора

Доступны только пользователям, чьи Telegram ID перечислены в `TELEGRAM_ADMIN_IDS`.

* `/admin` --> Справка по админ-командам.

* `/pending` --> Список заявок, ожидающих решения.

* `/approve` --> Подтвердить заявку по номеру.

```text
/approve 12
```

* `/reject` --> Отклонить заявку по номеру.

```text
/reject 12
```

При создании заявки через `/book` администраторы получают уведомление с
кнопками «Одобрить» и «Отклонить». Пользователь получает сообщение о решении.
До одобрения заявка имеет статус `pending` и блокирует аудиторию так же, как
одобренная.

## Сборка

Перед сборкой wheel-пакета doit автоматически компилирует файлы переводов
(задача `i18n_compile`).

Собрать wheel-пакет:

```bash
python3 -m doit wheel
```

Собрать source distribution:

```bash
python3 -m doit sdist
```

Собрать оба архива:

```bash
python3 -m doit build
```

Обновление каталогов переводов после изменения строк в коде:

```bash
python3 -m doit i18n_extract
python3 -m doit i18n_update
python3 -m doit i18n_compile
```

Очистить сгенерированные файлы сборки и покрытия:

```bash
python3 -m doit cleanup
```
